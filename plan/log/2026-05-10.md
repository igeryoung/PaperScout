## Phase 2 — Source collection closed

- Built three source clients in `src/server/sources/{arxiv,openreview,huggingface}.ts`. Each splits a pure parser (consumes raw XML/JSON, returns `Candidate[]` validated by `CandidateSchema`) from a `fetch*` wrapper that takes an injected `fetch` so tests don't hit live APIs. arXiv is parsed with `fast-xml-parser` (newly added); pdfUrl is read from the `<link type="application/pdf">` href, not by string-replacing `abs/`→`pdf/` (URL pattern can change). OpenReview converts `pdate` ms-epoch → ISO date and uses the v2 `notes/search` endpoint. HuggingFace `daily_papers` derives an arXiv pdfUrl + `additionalSources` ARXIV ref when `paper.id` matches `^\d{4}\.\d{4,5}$`, and applies a permissive CV keyword filter (LLM screening downstream is the safety net).
- `src/server/sources/types.ts` re-exports `Candidate as CandidateRecord` from `src/server/schema/candidate.ts`. The Phase 2 README's standalone shape with `Date publishedDate` and `rawMetadata` was discarded; the existing zod schema (with ISO-string dates and `additionalSources[]`) is the contract.
- Aggregator `src/server/sources/index.ts#collectFromAllSources`: `Promise.allSettled` over the three clients, within-batch dedup keyed by arXiv id / OpenReview id / normalized title with priority ARXIV > OPENREVIEW > HUGGINGFACE, then quotas 15/10/5 with overflow refill, trim to 30. Per-source counts logged.
- Pipeline `src/server/pipeline/{collect,persist,runner}.ts`: `persist` mirrors `scripts/ingest.ts:111-222` plus a `seenPaperId` guard against fuzzy collisions in one batch. `runner.startRun()` creates the DailyRun row (`ingestSourceDir: null` for API runs); `runCollectionInBackground` does collect → persist → setStatus(COMPLETED|FAILED), logged but never thrown.
