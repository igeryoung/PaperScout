# Log — 2026-05-11

## Phase 2.5 — Prompt harness closed

- Built an offline harness around the `evaluate-papers` Claude Code skill. No app-level Anthropic SDK, no DB, no UI — fixture builder + per-record schema check + soft-bound checker + regression-snapshot writer. Per the locked architecture decision, prompt execution stays in the skill; the harness validates output and freezes references.
- Authored 5 fixtures (`scripts/prompt-eval/fixtures/F{1..5}/{metadata,bounds}.json`) covering the quality spectrum: F1 SAM (2304.02643, strong novelty + rigor), F2 EfficientFormerV2 (2212.08059, incremental tweak), F3 ViT-22B (2302.05442, famous lab + thin novelty), F4 SeaThru-NeRF (2304.07743, unknown lab + strong method), F5 The Dawn of LMMs / GPT-4V (2309.17421, hype-driven + weak claims). Each metadata.json carries a `_fixture` provenance block (`metadataSourceUrl`, `authoredAt`, `frozen: true`) and a verbatim arXiv abstract — no paraphrases, no fabricated text.
- `scripts/prompt-eval/lib.ts` is the single source of truth: `BoundsSchema` (zod), `FixtureManifestSchema`, `loadFixtures`, `buildCandidateMap` (mirrors `scripts/ingest.ts:65-79` including additionalSources), `resolveJoinKey` / `recomputeTotal` / `applyBounds` / `checkRecordSchema` / `summarize`. All public functions take raw `unknown` so a single malformed record cannot blank the diagnostic report. Per-record validation uses `EvaluationSchema.safeParse` rather than the file-level array schema.
- BoundsSchema uses `z.partialRecord(BoundedScoreKey, NumericRange)` (zod v4) — `z.record` with an enum key requires *all* enum keys in v4 and would reject any partial bounds. Caught at first smoke run.
- `scripts/prompt-eval/build-fixture-run.ts` emits `candidates.json` + `fixtures-manifest.json` atomically into `scripts/prompt-eval/runs/<YYYY-MM-DD-HHMM>/`. Manifest is a first-class output (`fixtureId`, `primaryKey`, `allJoinKeys[]`, `metadataPath`, `boundsPath`) — the checker reads only the manifest, never re-derives from `fixtures/`.
- `scripts/prompt-eval/check-evaluations.ts` validates schema per-record, applies bounds, prints schema/scores/bounds tables, the actual ranking, and three coarse flags (`f1InTop2`, `f5InBottom2`, `f4NotLast`). Exits 0 iff schema-valid for all + ≥4/5 bounds + zero unmatched joinKeys.
- `scripts/prompt-eval/normalize-evaluations.ts` reorders keys deterministically, recomputes `scores.total` only in the normalized copy, and writes `reference/normalized/F<n>.json` per fixture for regression diffs.
- `package.json` gained `prompt:fixtures`, `prompt:check`, `prompt:normalize` (no env-file flag — no DB). `.gitignore` adds `/scripts/prompt-eval/runs/` and `/scripts/prompt-eval/fixtures/**/paper.pdf` (per the user's "no PDFs in git" choice; local-PDF support in the skill is deferred — see below).
