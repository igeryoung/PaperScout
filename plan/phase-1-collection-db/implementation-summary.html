Phase 1 實作摘要

Phase 1 把論文收藏資料庫的基礎變成可實際使用的功能：加入種子資料、 建立唯讀 Library 頁面、補上列表查詢，並更新計畫狀態。大部分 schema、repo、dedup 工作已在 Phase 0.5 完成，這一階段主要是把它們接到可操作的介面上。

9 Prisma 實體已建模：papers、runs、sources、evaluations、feedback、tags、code links、duplicates。
5 種子論文橫跨 arXiv、OpenReview、Hugging Face 三個來源。
1 新增 /library 論文庫頁面。
29 實作後通過的單元測試數。

這次改了什麼

種子資料

prisma/seed.ts 會寫入五篇範例論文與對應的來源記錄。 它具備冪等性，重跑不會產生重複的種子論文。

Library 查詢

papersRepo.listLibrary() 回傳 UI 需要的唯讀欄位： 標題、作者、來源、發表日期、收錄日期。

Library 頁面

src/app/library/page.tsx 以簡單表格顯示已收錄論文； 如果資料庫是空的，也會顯示空狀態提示。

計畫狀態

plan/STATE.md 與 Phase 1 README 已標記 Phase 1 關閉， 並把下一步指向 Phase 2 的來源收集工作。

執行流程

1 執行 Seed npm run db:seed 將範例論文寫入 Postgres。
2 建立論文 當 fingerprint 尚不存在時，每筆種子資料會建立一列 canonical Paper。
3 掛上來源 每篇論文都會建立對應的 PaperSource，來源可能是 arXiv、OpenReview 或 Hugging Face。
4 讀取 Library Server Component 呼叫 papersRepo.listLibrary({ limit: 50 })。
5 渲染表格 瀏覽器在 /library 顯示已收錄論文清單。
